## 1. 플로우차트 (Flowchart)

가장 기본이 되는 순서도다. 조건 분기와 반복 흐름을 표현할 때 쓴다.

```mermaid
flowchart TD
    A[시작] --> B{로그인 정보 확인}
    B -->|성공| C[대시보드 이동]
    B -->|실패| D[에러 메시지 표시]
    D --> E[다시 입력]
    E --> B
    C --> F[종료]
```

---

## 2. 시퀀스 다이어그램 (Sequence Diagram)

객체 간에 시간 순서대로 오가는 메시지를 표현한다. API 통신 설명에 자주 쓴다.

```mermaid
sequenceDiagram
    participant U as 사용자
    participant B as 브라우저
    participant S as 서버
    U->>B: 로그인 버튼 클릭
    B->>S: POST /login 요청
    S->>S: 인증 처리
    S-->>B: JWT 토큰 반환
    B-->>U: 대시보드 화면 표시
```

## 3. WBS (작업 분해 구조)

머메이드에 WBS 전용 문법은 없어서 `graph TD` 트리로 계층을 표현하는 방식이 가장 WBS답게 나온다. 번호 체계를 넣어주면 실제 WBS 느낌이 살아난다.

```mermaid
graph TD
    P[웹서비스 구축 프로젝트]
    P --> A[1. 기획]
    P --> B[2. 개발]
    P --> C[3. 배포]
    A --> A1[1.1 요구사항 분석]
    A --> A2[1.2 화면 설계]
    B --> B1[2.1 프론트엔드]
    B --> B2[2.2 백엔드]
    B --> B3[2.3 DB 설계]
    C --> C1[3.1 서버 세팅]
    C --> C2[3.2 테스트]
```

참고로 `mindmap` 문법으로도 WBS 비슷하게 그릴 수 있는데, 네모 박스 계층 구조가 필요하면 위처럼 `graph TD`가 더 적합하다.

## 4. 간트 차트 (Gantt Chart)

프로젝트 일정을 시간축으로 보여준다. WBS로 나눈 작업을 그대로 일정으로 옮기면 이어서 설명하기 좋다.

```mermaid
gantt
    title 웹서비스 구축 일정
    dateFormat YYYY-MM-DD
    section 기획
    요구사항 분석      :a1, 2025-08-01, 5d
    화면 설계          :a2, after a1, 4d
    section 개발
    프론트엔드         :b1, after a2, 10d
    백엔드            :b2, after a2, 12d
    section 배포
    테스트            :c1, after b2, 5d
    출시             :milestone, after c1, 0d
```

## 5. ER 다이어그램 (Entity Relationship)

데이터베이스 테이블 간 관계를 표현한다. 카디널리티(`||--o{`) 기호를 설명하면 학생들이 관계형 모델을 이해하는 데 도움이 된다.

```mermaid
erDiagram
    회원 ||--o{ 주문 : 생성
    주문 ||--|{ 주문상품 : 포함
    상품 ||--o{ 주문상품 : 등록
    회원 {
        int 회원ID PK
        string 이름
        string 이메일
    }
    주문 {
        int 주문ID PK
        int 회원ID FK
        date 주문일자
    }
    상품 {
        int 상품ID PK
        string 상품명
        int 가격
    }
```
